이 프로젝트의 **docs/ 4문서 체계**를 현재 상태로 업데이트합니다.

## 업데이트 순서 (반드시 아래 순서대로 진행)

### 0. docs/plan.md 자동 체크 (task.md 완료 항목 동기화)

`docs/task.md`를 읽어 완료(✅) 표시된 단계를 파악한 후 `docs/plan.md`와 동기화합니다:

1. `docs/task.md`에서 완료 표시된 단계 이름 / 파일명 / 단계 ID 추출
2. `docs/plan.md`에서 해당 항목을 검색하여 `[ ]` → `[x]` 변경
3. Phase의 모든 항목이 `[x]`로 완료된 경우 → Phase 제목에 `✅ 완료` 추가
4. 자산 현황 대시보드에서 수량이 변경된 항목 업데이트
   - Skills / Agents / Hooks / Commands 수량이 바뀐 경우에만 수정

> **목적:** task.md 완료 기준으로 plan.md를 자동 동기화하여 수동 누락 방지

### 1. docs/history.md 업데이트

이 문서는 가장 중요합니다. 다음 세션에서 맥락을 이어받을 수 있도록 작성하세요:

**업데이트 내용:**
- **현재 상태 요약** 섹션: 현재 Phase, 직전 완료 작업, 다음 작업 업데이트
- **자산 현황 스냅샷**: 변경된 수량 반영 (Skills/Agents/Commands/Hooks 수)
- **작업 이력** 섹션: 이번 세션에서 완료한 작업을 날짜와 함께 추가
- **다음 세션에서 이어갈 작업** 섹션: 다음 Phase 또는 미완료 작업 상세 기록
- `Last updated` 날짜 갱신

### 2. docs/checklist.md 업데이트

완료된 항목에 `[x]` 표시:
- 완료된 Step의 모든 하위 항목 체크
- 새로운 Phase 시작 시 새 체크리스트 항목 작성

### 3. docs/task.md 업데이트

현재 Phase 완료 시:
- 완료된 Step 항목 표시
- 다음 Phase 준비 안내로 내용 교체

현재 Phase 진행 중인 경우:
- 완료된 Step 표시
- 잔여 작업 목록 업데이트

### 4. docs/plan.md 업데이트 (Phase 완료 시에만)

Phase 완료 시:
- 해당 Phase 항목에 `✅ 완료` 표시
- `[x]` 체크박스 전체 체크
- 자산 현황 대시보드 수량 업데이트

### 5. base_code_plan.md 업데이트 (자산 변경 시에만)

자산 수량 변경 시:
- 해당 섹션 수량 수정 (예: `commands/ ← 슬래시 커맨드 라이브러리 (3개)` → `(4개)`)
- 새 자산 행 추가 (해당 시)
- `Last updated` 날짜 갱신
